1.6 lsof: 从系统获得与网络连接相关的信息

(http://www.wenwenyun.com/opration/linux/2014/1122/2922.html)

  • 使用-i显示所有连接
  • 使用-i 6仅获取IPv6流量
  • 仅显示TCP连接（同理可获得UDP连接）: lsof -iTCP
  • 使用-i:port来显示与指定端口相关的网络信息: 或者，你也可以通过端口搜索，这对于要找出什么阻止了另外一个应用绑定到指定端口实在是太棒了。
  • 使用@host来显示指定到指定主机的连接: lsof -i@172.16.12.5
  • 使用@host:port显示基于主机与端口的连接: lsof -i@172.16.12.5:22
  • 找出监听端口: lsof -i -sTCP:LISTEN
  • 找出已建立的连接: lsof -i -sTCP:ESTABLISHED
  • 使用-u显示指定用户打开了什么: lsof -u daniel
  • 显示除指定用户以外的其它所有用户所做的事情: lsof -u ^daniel
  • 杀死指定用户所做的一切事情: kill -9 `lsof -t -u daniel`
  • 显示与指定目录交互的所有一切: lsof /var/log/messages/
  • 显示与指定文件交互的所有一切: lsof /home/daniel/firewall_whitelist.txt

(link) working with vim, depends on tern (refer to Tagbar)

3.5.1 Intallation

3.5.2 Usage with vim

If you'd like to take a JavaScript project and generate a tags file that Vim can parse, you can use the below command. It searches your directory for any .js files, excluding ./nodemodules, formats the tags correctly for Vim and outputs them into tags. 

find . -type f -iregex .*\.js$ -not -path "./node_modules/*" -exec jsctags {} -f \; | sed '/^$/d' | sort > tags

If you would like tags generated for files within nodemodules/, just remove the -not -path "./nodemodules/* part of the command: 

find . -type f -iregex .*\.js$ -exec jsctags {} -f \; | sed '/^$/d' | sort > tags

4 Technology Trends

4.1 一秒启动Linux   linux automobile booting

在欧洲举行的嵌入式Linux会议上，德国公司Linutronix 的Jan Altenberg介绍了如何一秒启动Linux。一秒启动对于汽车等应用环境十分有用。使用Atmel AT91 ARM9 SoC和NAND flash存储器，运行Angstrom Linux的标准启动时间是11秒，简单优化后可以降低到8秒，深入广泛的修改包括内核配置修改和命令行调整，可以将启动时间减少到 656毫秒。具体细节可浏览他的演讲报告（PDF）。

5.1.1 How to create a bootable USB stick on OS X

(link)

  1. 1. Download ubuntu desktop

    go to http://www.ubuntu.com/download/desktop

  2. 2. Open the terminal
  3. 3. Convert the .iso file to .img

    use the convert option of hdiutil 

    hdiutil convert -format UDRW -o ~/path/to/target.img ~/path/to/ubuntu.iso
  4. 4. Run disktuil to get the current list of devices
    diskutil list
  5. 5. Insert your flash media
  6. 6. Run diskutil again and determine the device node assigned to your flash media

    [ME: /dev/disk1]

  7. 7. Run diskutil to unmount your flash media
    diskutil unmountDisk /dev/disk1
  8. 8. Write .img to your flash medi
    sudo dd if=/path/to/downloaded.img of=/dev/rdisk1 bs=1m

    (replace /path/to/downloaded.img with the path where the image file is located; for example, ./ubuntu.img or ./ubuntu.dmg).

    • Using /dev/rdisk instead of /dev/disk may be faster
    • If you see the error dd: Invalid number '1m', you are using GNU dd. Use the same command but replace bs=1m with bs=1M
    • If you see the error dd: /dev/diskN: Resource busy, make sure the disk is not in use. Start the 'Disk Utility.app' and unmount (don't eject) the drive
  9. 9. Eject your flash media
    diskutil eject /dev/disk1
  10. 10. Restart your MacBook and press Alt/Option key while it is restarting to choose the USB stick.
