1.6 p 方法

  • 数值结果和字符串结果会以不同的形式输出
    puts "100"  #=> 100
p "100" #=> "100"
p 100  #=> 100
  • 特殊字符不会转义，原样输出
  • -> 原则上，p 方法是提供给编程者使用的: 用于确认程序执行情况

1.7 中文的输出

  • Ruby 程序的编码方式，是通过在程序的首行代码添加注释 # encoding: 编码方式 来指定的（编码的规则称为 encoding）。我们称这个注释为魔法注释（magic comment）。

8.5 Module 是什么

Ruby 的特色功能之一。

Class 和 module 的不同之处：

  • Class 表现的是事物的实体（数据）及其行为（处理）。[ME: 即 type, "is-a"]
  • Module 只表现事物的行为部分。 [ME: 即 role，"behave-like-a", 参考 poodr]
  • Module 不能有实例。
  • Module 不能被继承。

8.6 Module 的典型用法

8.6.1 namespace

对方法、常量、类等名称进行区分及管理的单位 -> 通过在 module 内部定义名称，就可以解决命名冲突的问题。

语法：~modulename.methodname~。 

p FileTest.exist?("/usr/bin/ruby")  # check if a file exists
p FileTest.size("/usr/bin/ruby")

如果名字没有冲突，引用的时候就可以省略 module 名。

使用 include 可以吧 module 里的名字合并到当前的命名空间。 

ap Math::PI
ap Math.sqrt(2)

include Math  # 包含 Math module
p PI
p sqrt(2)

通过把相关功能汇总到一个 module 中，就可以集中管理相关的命名。

8.6.2 利用 mix-in 扩展功能

就是将 module 混合到 class 中：在定义 class 时使用 include 把 module 里的名字合并到类中使用。

Mix-in 可以更加灵活地解决下面的问题：

  • 虽然两个 class 有相似的功能，但是不希望把他们作为相同种类（不希望有共同的 superclass）来考虑 [ME: role!]
  • Ruby 不支持 superclass 的多重继承，因此无法对已经继承的 class 添加共同的功能（参见 8.8 节）

Share code (also share role) using module: 

module MyModule
  # common methods
end

class MyClass1
  include MyModule
  # methods specific to MyClass1
end

class MyClass2
  include MyModule
  # methods specific to MyClass2
end

8.9 面向对象程序设计

8.9.1 什么是 object

程序语言将程序处理的主体作为 object 来考虑。

一般情况下，程序语言的处理主体是数据。-> 面向对象的语言中的对象就是指数据（或数据的集合）及操作该数据的方法的组合。 

ap (3.14).to_s
ap (3.14).round
ap (3.14).ceil
ap (3.14).to_i

开发大型应用的时候，需要将大量的数据整合在一起并根据一定的规则进行处理，也就是作为对象来看待。例如，利用 Ruby 库里的 Net::HTTP，Net::POP 等类，可以轻松地编写网络应用。

8.9.2 面向对象的特征
封装（encapsulation）

使 object 管理的数据不能从外部直接访问，数组操作必须通过调用 object 的方法来完成 [ME: 稳定的 interface]。防止把非法数据设置给 object 从而产生异常。

需要编写不会让 object 内容部产生异常的方法，定义方法时就应该考虑如何避免错误的发生。

Ruby 对象默认情况下是被强制封装的，因此无法从外部直接访问 Ruby 对象的实例变量。建议只有在必要时才使用 attraccessor 这样的方法公开实例变量。

封装的另一个好处是可以隐藏对象内部数据处理的具体细节 -> [ME: 封装的一个作用：隐藏变化和复杂性]

多态（polymorphism）

对象知道其数据如何处理，也就是说，各个对象都有自己独有的消息解释权。一个方法名属于多个对象（不同对象的处理结果也不一样）-> 多态性。例如 Object、String、Float 类的 tos 方法处理方法不同。

8.9.3 Duck Typing

中心思想：对象的特征不是由其种类（class 及继承关系）决定的，而是由对象本身具有什么样的行为（拥有什么样的方法）决定的。

Ruby 中的变量不限定类型。

  • 缺点：因此在程序运行之前无法知道变量指定的对象的方法表用是否正确。增加了程序运行前检查错误的难度。
  • 好处：非常容易使没有明确继承关系的对象之间的处理变得通用。

  end

  end
end

  • 系统错误
  • 程序错误

  • 忽略错误
  • 回复错误发生前的状态

10.2 异常处理

程序运行时发生错误就会产生异常，异常发生后，程序会暂时停止运行，并寻找是否有对应的异常处理程序。如有则执行，如没有，程序就会显示信息并终止运行。

错误信息的格式： 

文件名:行号:in 方法名:错误信息(异常类名)
      from 文件名:行号:in 方法名

Ruby 中的异常处理： 

begin
  a()
  b()
  c()
rescue
  错误处理  # a，b，c中发生异常
end

异常处理的优点：

  • 程序不需要逐个确认处理结果，也能自动检查出程序错误
  • 程序会报告发生错误的位置，便于排查错误
  • 正常处理与错误处理的代码可以分开书写，便于阅读

10.3 异常处理的写法

begin
  可能会发生异常的语句
rescue => 引用异常对象的变量  # 可省略 『=> 引用异常对象的变量』
  发生异常处理时的处理
end

异常及其相关信息都作为对象处理，在 rescue 后指定指定变量名则可以获取异常对象。即使不指定变量名，也可以通过特殊变量 $! 获取异常对象。推荐明确写出变量名以提高可读性。

异常发生时被自动赋值的变量：

  • $!: 最后发生的异常（异常对象）
  • $@: 最后发生的异常位置信息

异常对象的方法：

  • class
  • message
  • backtrace: 异常发生的位置信息（ $@ = $!.backtrace ）

10.4 后处理

使用 ensure 关键字来定义不管是否发生异常都需要执行的处理。

  有可能发生异常的处理

  发生异常后的处理
ensure
  不管是否发生异常都希望执行的处理
end

例如： 

# copy file from one place to another

  src = File.open(from)  # 如果此处发生异常将返回给调用者
  begin
    dst = File.open(to, "w")
    data = src.read
    dst.write(data)
    dst.close
  ensure
    src.close  # src 打开后，不管能否正常继续执行，都必须关闭 src
  end
end

11. Block

Ruby 的重点之一：如何灵活地使用 block。

11.1 什么是 block

block：在调用方法时，能与参数一起传递的、多个处理 [ME:语句]的集合。

Block 的执行次数由调用它的方法决定。

Block 的调用方式： 

object.method_name(argument_list) do |block_var|
  statements
end

or 

object.method_name(argument_list) { |block_var|
  statements
}

Block 变量就是在执行 block 时，由调用它的方法传进来的参数。

11.2 Block 的使用方法

11.2.1 Loop

Ruby 中常使用 block 来实现循环。

在接收 block 的方法中，实现了循环处理的方法称作 iterator（例如 each 方法就是一个典型的 iterator） 

sum = 0
outcome = {'canjiafei' => 1000, "guajianfei" => 1000, "welcome" => 4000}
outcome.each do |pair|
  sum += pair[1]  # pair[1] 对应 value，pair[0] 对应 key
end
puts "total: #{sum}"

>>>>
total: 6000

接收 block 变量时，也适用多重赋值规则： 

sum = 0
outcome = {'canjiafei' => 1000, "guajianfei" => 1000, "welcome" => 4000}
outcome.each do |item, price|
  sum += price
end
puts "total: #{sum}"

File 类的 iterator 有：

  • eachline
  • eachchar: 逐字符处理
  • eachbyte: 逐字节处理

其他对象的情况类似：eachXX。

11.2.2 隐藏常规处理

除了循环，还常用于确保后处理被执行。 

File.open("sample.txt") do |file|
  file.each_line do |line|
    print line
  end
end

注意上例不需要调用 close 方法，在其内部进行了如下的处理： 

file = File.open("sample.txt")
begin
  file.each_line do |line|
    print line
  end
ensure
  file.close
end

推荐用法: File.open 方法使用 block 时，block 内部处理完毕跳出方法前，文件会被自动关闭。这样就可以一方面减少代码量，一方面防止忘记关闭文件等错误的发生。

17.1.1 Standard IO

程序在启动后会预分配3个 IO 对象：

17.3 文件指针

处理文本数据时，一般以行为单位。只有当读取到换行符才能知道行的长度-》如果要读取第 100 行的数据，就要读取这 100 行的全部数据。如果修改了数据，行的长度也会变化，文件后面的数据就都要做出修改。

为了提高读取效率，可以将文件分成固定长度的文件快，来直接访问某个位置的数据 -》 虽然可以访问任意位置的数据，但不能处理超过指定长度的数据。

用文件指针（file pointer）或当前文件偏移量（current file offset）来表示 IO 对象指向的文件的位置。读取文件时，file pointer 会自动移动。我们也可以通过使 file pointer 指向任意位置来读写数据。

io.pos, io.pos=(position)

pos 方法可以获取 file poiter 的当前位置。使用 pos= 方法改变 file pointer 位置。 

# hello.txt 中的内容为 "Hello, Ruby.\n"
File.open("hello.txt") do |io|
  p io.read(5)  #=> "Hello"
  p io.pos  #=> 5
  io.pos = 0
  p io.gets  #=> "Hello, Ruby.\n"
end
io.seek(offset, whence)

移动文件指针。offset 是指定位置的整数，whence 用于指定 offset 如何移动。

whence 的值：

  • IO::SEEKSET：移动到 offset 指定的位置
  • IO::SEEKCUR：将 offset 视为相对于当前位置的便宜量
  • IO::SEEKEND：将 offset 视为相对于文件末尾的偏移量
io.rewind

将文件指针返回到文件开头。lineno 方法返回的行编号变成 0。 

# hello.txt 中的内容为 "Hello, Ruby.\n"
File.open("hello.txt") do |io|
  p io.gets  #=> "Hello, Ruby.\n"
  io.rewind
  p io.gets  #=> "Hello, Ruby.\n"
end
io.truncate(size)

按照 size 指定大小截断文件。 

io.truncate(0)  # 将文件大小置为 0
io.truncate(io.pos)  # 删除当前文件指针以后的数据

17.4 二进制模式与文本模式

不同平台下换行符不同（参考 [第14章]）。为了确保程序的兼容性，会将 string 中的 『\n』转换为当前 OS 的换行符并输出。此外，在读取的时候也会将实际的换行符转换为『\n』。

当需要确定文件大小进行 IO 处理时，或直接使用从其他平台拷贝的文件时，如果进行换行符转换，可能会引发问题。

为了解决上述问题，Ruby 提供了不进行换行符转换的方法：换行符处理的前提是以行为单位做 IO 处理，需要转换时称为文本模式。不需要转换时称作二进制模式。

io.binmode

新的 IO 对象默认是文本模式，使用 binmode 方法将其变更为二进制模式。 

File.open("foo.txt", "w") do |io|
  io.binmode
  io.write "Hello, world.\n"
end

这样就可以既不用转换换行符，又能得到与文件中一模一样的数据。

Note 已经转换为二进制模式的 IO 对象无法再次转换为文本模式。

17.5 缓冲

操作 IO 对象进行输出时（如 write、print 等方法），程序内部会使用 buffer 来保存临时生成的数据副本，积累一定的数据后，再做实际的输出处理，然后清空 buffer。-》 buffering。

stderr 完全不使用 buffering，因此 stdout 与 stderr 混合使用时，实际输出的顺序可能会与程序中的顺序不一致。

Buffering 可以提高输出效率，但有时我们希望可以立即显示执行结果 -》 使用下面的方法来同步数据的操作与输出：

io.sync, io.sync=(state)

使用 io.sync = true ，程序向 buffer 中写入数据时会自动调用 flush 方法。 

$stdout.sync = true
$stdout.print "out1"

Example:

$stdout.sync = false

19. Encoding 类

19.1 Ruby 的编码与字符串

即使在同一个程序中，有时候 input/output 的字符编码也可能不一样。例如，输入是 UTF-8，而输出是 ShiftJIS，需要经过适当的转换。

Ruby 的每个 string 对象都包含『string 数据本身』以及『该数据的字符编码』两个信息。

创建 string 一般有两种方式：

  • 脚本中使用字面量
  • 从程序外部（file、console、网络等）获取数据

String 的获取方式决定了它的编码方式。截取 string 的某部分，或连接多个 string 生成新 string 时，编码会继承原有 string 的编码。

程序向外部疏忽 string 时，必须指定适当的编码。

Ruby 会按照以下信息决定 string 对象的编码，或者在 input/output 处理时转换编码：

  • 脚本编码，参考 [19.2 节]
  • 内部编码与外部编码，参考 [19.5 节]

19.2 脚本编码与魔法注释

Ruby 脚本的编码就是通过在脚本开头使用魔法注释指定的。

脚本编码（script encoding）。脚本中的 string 、regex 的字面量会依据脚本编码进行解释。

Ruby 在解释脚本前，会先读取魔法注释（magic comment）来决定脚本编码。必须写在脚本的首航（第一行以 #!~ 开头时，应写在第二行）。

例子：指定 utf-8 编码 

# encoding: utf-8

Tip 在 Unix 中，赋予脚本执行权限后，就可以直接执行脚本。这时可以在文件开头用 #! 命令的路径形式来指定执行脚本的命令。例如可以这么写 

#! /usr/bin/ruby

然后就能用 >脚本名 的形式执行脚本了。

Tip 为了兼容 Emacs、VIM 等编辑器的编码指定方式，可以这么写： 

# -*- coding: utf-8 -*-  # for Emacs
# vim:set fileencoding=utf-8  # for VIM

程序代码的编码会严格检查是否与脚本编码一致。

Ruby 2.0 中，没有魔法注释时默认脚本编码为 UTF-8。

有时仅使用魔法注释是不够的。例如，使用特殊字符 \u 创建 string 后，即使脚本编码不是 UTF-8，其生成的 string 也一定是 UTF-8。 

# encoding: EUC-JP
a = "\u3042\u3044"
puts a  # 日文
p a.encoding  #=> UTF-8

此时必须使用 encode! 方法明确进行编码转换： 

# encoding: EUC-JP
a = "\u3042\u3044"
a.encoding!("EUC-JP")
p a.encoding  #=> EUC-JP

19.5.3 编码的作用

只要使用的环境与实际使用的数据编码一致，就不需要考虑编码转换问题。

另一方面，如果执行环境与数据编码不一致，就需要在代码中有意识地处理编码问题。

输出时编码的作用

外部编码影响 IO 的写入（输出）。

输出时，会基于每个 string 的原有编码和 IO 对象的外部编码进行转换（因此输出用的 IO 对象不需要指定内部编码）。

如果没有设置外部编码，或者 string 的编码与外部编码一致，则不需要进行编码转换。

需要进行转换的时候，如果输出的 string 编码不正确（例如实际上是日语字符串，编码却是中文），或者是无法转换的编码组合（例如用于日语与中文的编码），此时程序将会抛出异常。

总结：

  • 外部编码未设定时不进行转换
  • 把 string 的编码转换为外部编码
  • 无法转换时抛出异常
输入时编码的作用

稍微复杂一点。

  • 如果没有设置外部编码，则会使用 Encoding.default_external 的值作为外部编码。
  • 设置了外部编码，但没有设置内部编码时，则会把要读取 string 的编码设置为 IO 对象的外部编码。这种情况下不会进行编码转换，而是将文件、console 输入的数据原封不动地保存为 String 对象。
  • 最后，外部编码和内部编码都设定的时候，则会执行由外部编码转换为内部编码的处理。输出与输出的情况一样，在编码转换过程中如果数据格式或编码组合不正确，程序会抛出异常。

总结：

  • 内部编码未设定时把外部编码设置给 string
  • 外部编码未设定时使用 Encoding.defaultexternal
  • 设定了内部编码时从外部编码进行转换
  • 无法转换时抛出异常

GB 18030，全称：国家标准GB 18030-2005《信息技术 中文编码字符集》，是中华人民共和国现时最新的内码字集，是GB 18030-2000《信息技术 信息交换用汉字编码字符集 基本集的扩充》的修订版。与GB 2312-1980完全兼容，与GBK基本兼容，支持GB 13000及Unicode的全部汉字，共收录汉字70244个。 GB 18030主要有以下特点：

def ls_t(path, flag = "m")
  files = {}
  Dir.open(path) do |dir|
    begin
      dir.each do |name|
        next unless name != "." && name != ".."
        stat = File.stat(File.join(path, name))
        case flag
        when "c"
          time = stat.ctime
        when "a"
          time = stat.atime
        when "m"
          time = stat.mtime
        else
          raise "unknown option: " + flag
        end
        files[name] = time.strftime "%Y-%m-%d %H:%M:%S"
      end
    rescue => ex
      p ex.message
    end
  end
  files.sort do |a, b|
    a[1] <=> b[1]
  end.each do |key, value|
    printf "%s  %s\n", value, key
  end
end
